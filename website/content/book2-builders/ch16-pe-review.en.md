## ✨ Magic Spell #16 — Complete Prompt Engineering Review

**Everything you learned in Book 2 — all in one place:**

---

**1. Zero-Shot — Ask directly:**
```
Best for: Simple tasks, factual questions
Example: "What is the capital of Pakistan?"
```

**2. Few-Shot — Teach with examples:**
```
Best for: When you want a specific format or style
Example: Give 2-3 examples, then state the new task
```

**3. Chain of Thought — Step by step:**
```
Best for: Complex reasoning, maths, decisions
Example: "Think step by step and explain..."
```

**4. Role Prompting — Give a character:**
```
Best for: Specific tone, expertise, personality
Example: "You are a [expert]..."
```

**5. System Prompts — Permanent personality:**
```
Best for: Chatbots, API applications
Example: Define the full identity in the system field
```

---

**Prompt Engineering Decision Tree:**

```
Is the task simple?
├─ Yes → Try Zero-shot
└─ No → Is a specific format needed?
          ├─ Yes → Use Few-shot
          └─ No → Is reasoning needed?
                    ├─ Yes → Add Chain of Thought
                    └─ No → Try Role prompt
```

---

**Advanced Techniques (Preview for Book 3):**

```
• Prompt Chaining    → Output of one prompt into another
• Tool Use           → Giving AI real tools (calculator, search)
• Agent Prompting    → AI plans and executes on its own
• RAG                → Give AI your own data
```

---

**Common Mistakes and Fixes:**

| Mistake | Fix |
|---------|-----|
| Vague prompt | Be specific — format, length, tone |
| Everything in one long prompt | Break it into parts — have a conversation |
| Accepting the first answer | Refine — "improve it", "make it shorter" |
| Not saving the prompt | Save working prompts to a file! |
| Blaming the AI | Improve the prompt |
